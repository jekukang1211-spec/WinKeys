WinKeys for Mac

Windows 키보드 단축키를 Mac에서 그대로 사용하세요. Karabiner 없이, 가볍고 빠르게.

macOS 13+ 원클릭 설치 Karabiner 대체 메뉴바 앱

설치

사전 확인 Karabiner-Elements가 설치되어 있다면 먼저 제거해 주세요. 두 프로그램이 동시에 키보드 이벤트를 가로채면 충돌이 발생합니다.

방법 1: 자동 설치 (권장)

터미널을 열고 아래 명령어를 실행하세요:

cd ~/WinKeys
./install.sh

자동으로 빌드, /Applications에 복사, 실행까지 진행됩니다.

방법 2: 수동 설치

  1. 빌드

    터미널에서 실행:

    cd ~/WinKeys
./build.sh

  2. 앱 복사

    WinKeys.app을 /Applications 폴더로 복사합니다.

    cp -R WinKeys.app /Applications/

  3. 실행

    Launchpad 또는 Spotlight에서 WinKeys를 검색하여 실행합니다.

초기 설정 — 손쉬운 사용 권한

필수 권한 WinKeys는 키보드 입력을 변환하기 위해 손쉬운 사용(Accessibility) 권한이 반드시 필요합니다. 이 권한이 없으면 단축키 변환이 동작하지 않습니다.

WinKeys를 처음 실행하면 macOS가 자동으로 손쉬운 사용 권한을 요청합니다. 아래 순서대로 진행해 주세요:

  1. 시스템 설정 열기

    시스템 설정 > 개인정보 보호 및 보안 > 손쉬운 사용으로 이동합니다.

    또는 WinKeys 실행 시 자동으로 이 화면이 열립니다.

  2. 하단의 + 버튼 클릭

    목록 아래의 + 버튼을 클릭합니다.

    중요! 이미 목록에 WinKeys가 있더라도, 토글만 끄고 켜는 것으로는 부족할 수 있습니다. 확실하게 하려면:
    1. 목록에서 WinKeys를 선택 후 - 버튼으로 삭제
    2. 다시 + 버튼으로 추가
    이렇게 하면 앱을 새로 인식합니다.

  3. WinKeys 추가

    /Applications/WinKeys.app을 선택하고 열기를 클릭합니다.

  4. 토글 켜기

    목록에 추가된 WinKeys 옆의 토글을 켜짐으로 변경합니다. 잠금 해제(비밀번호 입력)가 필요할 수 있습니다.

  5. 메뉴바 확인

    화면 맨 위 상단 바(시계, Wi-Fi 아이콘이 있는 곳)에 W 아이콘이 나타나면 성공입니다.

권한이 적용되지 않을 때 손쉬운 사용에서 허용했는데도 동작하지 않으면, WinKeys를 완전히 종료 후 다시 실행해 보세요.
메뉴바의 W 아이콘 클릭 > WinKeys 종료 > 다시 실행

모드 전환

WinKeys는 두 가지 모드를 지원합니다:

W Windows 모드

기본 모드. 모든 단축키 변환이 활성화됩니다. Windows에서 익숙한 단축키를 그대로 사용할 수 있습니다.

M Mac 모드

모든 변환을 비활성화합니다. Mac 기본 단축키를 그대로 사용하고 싶을 때 전환하세요.

전환 방법

단축키동작
Ctrl+Alt+Shift+WWindows 모드로 전환
Ctrl+Alt+Shift+MMac 모드로 전환

메뉴바의 W/M 아이콘을 클릭하여 메뉴에서도 전환할 수 있습니다.

이 전환 단축키는 어떤 모드에서든 항상 동작합니다.

한/영 전환 키 설정

Windows 키보드에서 사용하던 한/영 전환 키(예: 오른쪽 Alt)를 Mac에서도 동일하게 사용할 수 있습니다.

설정 방법

  1. 설정 패널 열기

    메뉴바의 W 아이콘 클릭 > 언어 전환 키 설정... 선택

  2. 키 입력

    "언어 전환에 사용할 키를 눌러주세요" 창이 나타나면, 원하는 키를 누릅니다.

    예: 오른쪽 Alt, 오른쪽 Ctrl, 한/영 키 등

  3. 완료

    키를 누르면 자동으로 저장됩니다. 이후 해당 키를 누르면 한/영이 전환됩니다.

참고 설정한 키는 다음에 WinKeys를 실행할 때도 유지됩니다. 변경하고 싶으면 메뉴에서 다시 설정할 수 있습니다.

터미널 앱 예외

아래 터미널 앱에서는 Ctrl 키 리매핑이 자동으로 비활성화됩니다.

터미널에서 Ctrl+C는 프로세스 중단(SIGINT), Ctrl+Z는 일시정지 등 고유한 의미가 있기 때문입니다.

예외 처리되는 터미널 앱:

  • Terminal.app (기본 터미널)
  • iTerm2
  • Ghostty
  • Kitty
  • Alacritty
  • Hyper
  • Warp
참고 시스템 단축키(Alt+Tab, Win+D 등)와 내비게이션 키(Home/End)는 터미널에서도 동작합니다.

문제 해결

메뉴바에 W 아이콘이 안 보여요

  1. 앱이 실행 중인지 확인

    터미널에서 확인:

    ps aux | grep WinKeys | grep -v grep

    결과가 없으면 앱이 실행되지 않은 것입니다. open /Applications/WinKeys.app으로 실행하세요.

  2. 메뉴바 오른쪽 끝 확인

    시계, Wi-Fi, 배터리 아이콘이 있는 화면 맨 위 오른쪽 영역을 확인하세요. 아이콘이 많으면 W가 숨겨져 보이지 않을 수 있습니다.

단축키가 동작하지 않아요

  1. 손쉬운 사용 권한 재설정

    가장 확실한 방법은 - 버튼으로 삭제 후 + 버튼으로 다시 추가하는 것입니다:

    1. 시스템 설정 > 개인정보 보호 및 보안 > 손쉬운 사용 이동

    2. 좌측 하단 자물쇠 클릭 후 비밀번호 입력

    3. 목록에서 WinKeys 선택 > - 버튼 클릭하여 삭제

    4. + 버튼 클릭 > /Applications/WinKeys.app 선택 > 열기

    5. 토글을 켜짐으로 변경

  2. 앱 재시작

    메뉴바의 W 아이콘 클릭 > WinKeys 종료 후 다시 실행합니다.

    또는 터미널에서:

    killall WinKeys; sleep 1; open /Applications/WinKeys.app

  3. 현재 모드 확인

    메뉴바 아이콘이 W인지 M인지 확인하세요.

    M이면 Mac 모드로, 모든 변환이 비활성화된 상태입니다.

    Ctrl+Alt+Shift+W를 눌러 Windows 모드로 전환하세요.

업데이트 후 단축키가 안 돼요

앱을 업데이트(재빌드)하면 macOS가 새로운 앱으로 인식합니다. 손쉬운 사용 권한을 다시 설정해야 합니다.

반드시 - 로 삭제 후 + 로 다시 추가해 주세요. 토글만 끄고 켜는 것으로는 부족합니다.

Karabiner-Elements와 충돌

Karabiner가 설치되어 있으면 반드시 먼저 제거해 주세요:

sudo rm -rf /Applications/Karabiner-Elements.app \
  /Applications/Karabiner-EventViewer.app \
  '/Library/Application Support/org.pqrs'

제거 후 WinKeys를 재시작하세요.

특정 앱에서 Alt+F4가 안 돼요

Alt+F4는 Cmd+Q로 변환됩니다. macOS의 일부 시스템 앱(시스템 설정 등)은 Cmd+Q에 다르게 반응할 수 있습니다. 이것은 macOS 자체의 동작이며 정상입니다.

FAQ

Karabiner-Elements와 뭐가 다른가요?
WinKeys는 Windows 단축키 변환에 특화된 경량 앱입니다. Karabiner는 범용 키 리매핑 도구로 기능이 많지만 무겁고 설정이 복잡합니다. WinKeys는 설치 즉시 Windows 사용자에게 익숙한 단축키가 동작합니다.
터미널에서 Ctrl+C가 복사로 바뀌나요?
아니요. Terminal, iTerm2 등 터미널 앱에서는 Ctrl 키 변환이 자동으로 비활성화됩니다. Ctrl+C는 원래대로 프로세스 중단(SIGINT)으로 동작합니다.
잠시 원래 Mac 단축키를 쓰고 싶어요
Ctrl+Alt+Shift+M을 누르면 Mac 모드로 전환됩니다. 메뉴바 아이콘이 M으로 바뀌고 모든 변환이 비활성화됩니다.
다시 Windows 모드로 돌아가려면 Ctrl+Alt+Shift+W를 누르세요.
로그인할 때 자동으로 실행되게 하려면?
메뉴바의 W 아이콘 클릭 > 로그인 시 자동 실행을 체크하면 됩니다.
macOS 몇 버전부터 지원하나요?
macOS 13 (Ventura) 이상을 지원합니다.
손쉬운 사용 권한이 보안에 위험한가요?
손쉬운 사용 권한은 키보드/마우스 이벤트에 접근할 수 있는 권한입니다. WinKeys는 키보드 단축키 변환에만 이 권한을 사용하며, 키 입력을 기록하거나 외부로 전송하지 않습니다. 모든 처리는 로컬에서만 이루어집니다.
